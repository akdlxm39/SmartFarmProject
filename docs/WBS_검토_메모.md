# SmartFarmProject WBS 검토 메모

작성일: 2026-06-22
대상 파일: `references/SmartFarm_WBS_Simulation.xlsx`

## 총평
현재 WBS는 **초기 아이디어 단계의 End-to-End 풀스택/풀하드웨어 버전**을 기준으로 잘 정리되어 있다.
특히 역할분류(공통/로봇/AI/FE/BE/임베디드)와 큰 단계(기획 -> 환경구축 -> 코어구현 -> 통합 -> 테스트)는 유지해도 좋다.

다만, 최근 대화에서 프로젝트 방향이 더 단순화되었기 때문에 **그대로 가면 실제 구현 범위와 WBS가 어긋나는 항목**이 몇 개 있다.

## 유지해도 좋은 축
- WBS 및 기술스택 정리
- 3D CAD 모델링
- RoboDK 기반 사전 검증
- 데이터 기획 / AI 모델링 / 비전 파이프라인
- Django/MySQL 초기 세팅
- Vue/Chart.js 기반 대시보드 초안
- 통합 테스트 / 최종 산출물 정리

## 수정이 필요한 핵심 항목
### 1. 3열 분류 관련 표현
현재 시트에는 아래 표현이 들어가 있다.
- `RoboDK 활용 Dobot 수확/3열 분류 작업 영역(Reachability) 사전 검증`
- `Dobot 픽앤플레이스 기구학 제어 (수확 -> 카메라 -> 3열 분류/폐기 위치)`
- `RGB-D 객체 인식 기반 컨베이어 구동/정지 및 라벨 일치 3열 적재 로직`

하지만 현재 확정 방향은 다음과 같다.
- 1번 카메라가 메인 판정
- 불량 작물은 **Dobot이 바로 별도 상자에 투입**
- 정상 작물만 **컨베이어 끝 공용 상자 1개**로 이동
- **서보 분류기 제거**

즉, `3열 분류`, `3열 적재`, `분류기 동작`은 현재 방향과 맞지 않는다.

### 2. RGB-D 카메라 필수화
현재 WBS는 `RGB-D 카메라 및 압력센서 회로 구성`, `RGB-D 객체 인식 기반 컨베이어 구동/정지`가 들어가 있다.

그런데 지금 대화 기준으로 2번 카메라는
- 컨베이어 정중앙 상단
- 아래를 바라보는 일반 비전 카메라
- 존재/위치/흐름 확인
용도에 가깝다.

따라서 **RGB-D가 정말 필요한지 다시 확인**하는 게 좋다.
RGB-D가 필수가 아니라면 WBS 표현을 `상단 카메라 기반 컨베이어 흐름 감지` 정도로 낮추는 것이 맞다.

### 3. TurtleBot / 압력센서 / 배송 호출
현재 WBS에는
- `압력센서 회로 구성`
- `터틀봇 SLAM 기반 농장 맵핑`
- `압력센서 임계치 도달 시 터틀봇 배송 호출`
이 포함되어 있다.

이건 프로젝트를 꽤 크게 만든다.
현재까지의 핵심 흐름은 **수확 -> 1차 비전 판정 -> 불량 분리 / 정상 이송** 쪽이 더 선명하다.

따라서 TurtleBot은 다음 둘 중 하나로 정리하는 게 좋아 보인다.
- **MVP 제외**
- 또는 **후순위 확장 항목**으로 이동

3인 프로젝트 기준에서는 TurtleBot까지 동시에 안정적으로 가져가면 일정 리스크가 크다.

### 4. ROS Bridge / JSON 통신 규격
이건 유지해도 되지만, 지금은 `나중에 고려`라고 한 ROS-Django 연동 이슈가 있다.
따라서 WBS 상에서는
- 초반 필수 작업
보다는
- 통합 단계 작업
으로 미루는 게 더 자연스럽다.

## 추천 수정 방향
### 최소 수정안
현재 시트 틀은 유지하고, 아래 항목 표현만 바꾼다.

- `3열 분류` -> `정상/불량 분기 및 정상작물 공용 수거`
- `RGB-D 객체 인식` -> `상단 카메라 기반 컨베이어 존재/위치/흐름 확인`
- `서보 분류기` 관련 암시 -> 제거
- `터틀봇 배송` -> `후순위 확장` 또는 `선택 과업`

## 3인 프로젝트 기준 추천 우선순위
### 1차 핵심(MVP)
- Dobot 수확
- 1번 카메라 3방향 촬영
- 작물 3종 + 불량 판별
- 불량 즉시 분리
- 정상만 컨베이어 이송
- 2번 카메라 기반 흐름 확인
- 기본 웹 대시보드 / 로그

### 2차 확장
- ROS-Django 정교한 브리지
- 통계 고도화
- TurtleBot 물류
- 물/비료 분사 기능

## 결론
현재 WBS는 **뼈대는 좋고**, 그대로 버릴 필요는 없다.
다만 아래 4개는 수정 권장이다.

1. `3열 분류/적재` 표현
2. `RGB-D 필수` 가정
3. `서보 분류기` 전제
4. `TurtleBot 배송`의 우선순위

즉, **전체 틀은 유지 + 몇 개 핵심 항목만 현재 결정사항에 맞게 정리**하면 충분하다.
