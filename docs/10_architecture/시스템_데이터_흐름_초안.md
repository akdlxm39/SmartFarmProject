# SmartFarmProject 시스템 데이터 흐름 초안

작성일: 2026-06-22

## 1. 현재 확정된 핵심 원칙
- **1번 카메라가 메인 판정 카메라**다.
- 1번 카메라는 Dobot 석션컵에 흡착된 작물을 **120도 간격 3방향**으로 촬영한다.
- 1번 카메라는 **작물 종류 3종 분류 + 불량 여부(썩음 여부) 판별**을 함께 수행한다.
- **2번 카메라는 라인 흐름 확인용 보조 비전 카메라**다.
- 2번 카메라는 컨베이어 벨트 위 작물의 존재, 위치, 타이밍을 확인하고 컨베이어 구동을 보조한다.

## 2. End-to-End 데이터 흐름
1. 작물 위치에 대해 Dobot이 석션컵으로 작물을 흡착한다.
2. Dobot이 1번 카메라 촬영 위치로 이동한다.
3. 1번 카메라가 작물을 3방향(0도, 120도, -120도)으로 촬영한다.
4. 비전 모델이 3장의 이미지를 바탕으로 아래 2개 결과를 산출한다.
   - 작물 종류: 3종 중 1개
   - 품질 판정: 정상 / 불량(썩음)
5. 판정 결과를 제어 노드로 전달한다.
6. **불량 판정이 나오면 Dobot이 작물을 별도의 불량품 상자에 넣는다.**
7. **정상 판정이 나오면** Dobot이 작물을 컨베이어 벨트 시작점에 올린다.
8. 2번 카메라가 벨트 위 작물 존재 여부와 현재 위치를 확인한다.
9. 컨베이어 제어 노드가 작물 진입 시 벨트를 가동한다.
10. 정상 작물은 컨베이어 끝의 **하나의 공용 수거 상자**로 이동한다.
11. 각 단계 상태와 결과는 백엔드/DB 및 웹 관제 화면에 기록된다.

## 3. 카메라별 역할 정리
### 3.1 1번 카메라
입력:
- Dobot이 들고 있는 작물
- 3방향 촬영 이미지

출력:
- `crop_type`: 작물 종류 3종 중 하나
- `quality_status`: 정상 / 불량
- `inspection_id`: 개별 작물 추적용 ID

역할:
- 메인 분류/판정
- 다각도 기반 불량 검출
- 이후 라인 제어를 위한 기준 결과 생성

### 3.2 2번 카메라
입력:
- 컨베이어 위 작물의 실시간 영상

출력:
- `object_detected`: 작물 존재 여부
- `belt_position`: 현재 위치 또는 분류 구간 진입 여부
- `belt_exit_detected`: 벨트 끝 도달 여부

역할:
- 컨베이어 가동 보조
- 벨트 위 작물 흐름 모니터링
- 수거 상자 투입 전 위치/타이밍 확인
- 필요 시 예외 상황 감지(작물 미도착, 위치 어긋남 등)

## 4. 제어 로직 초안
### 4.1 Dobot 제어
- 모델/제어: Dobot Magician + ROS 2
- HOME/READY: `magician_ros2` homing 후 `(0, 0, 0)` 기준
- Dobot 배치 변경 시 TCP/실제 파지점 차이를 고려해 현재 pose를 읽고 좌표를 재보정
- 수확 위치: 고정 9개 좌표 중 하나를 `harvest_index`로 선택
- 작물 흡착
- 흡착 성공 여부는 우선 시간 기반 2초 대기로 처리
- 이동은 jump move 방식으로 z=100mm까지 상승 후 x/y 이동, 목표 z 하강 순서로 수행
- 1번 카메라 촬영 위치로 이동
- 촬영 위치: `(240, 0, 0)`
- 석션컵 회전으로 0도/120도/-120도 촬영 포즈 유지 후 0도로 복귀
- 현재 비전 미연동 상태에서는 각도별 촬영 완료/판정 대기 코드를 주석 처리하고, 각 촬영 포즈에서 2초 대기 후 다음 동작 진행
- 추후 비전 노드 연결 시 각도 도달 알림 후 비전 노드의 촬영 완료 응답 및 판정 완료 신호 수신으로 전환
- 불량 판정 시 불량품 상자 `(240, 140, 0)`에서 석션 OFF
- 정상 판정 시 컨베이어 시작 위치 `(160, -200, -10)`에 작물 배치
- 홈/대기 위치 `(150, 0, 100)mm`로 복귀

### 4.2 비전 판정
- 1번 카메라 3장 촬영
- 모델 추론
- 종류/불량 결과 생성
- 결과를 캐시 또는 메시지로 저장

### 4.3 컨베이어 제어
- 2번 카메라는 D435i를 사용하되, MVP에서는 RGB 프레임 + 고정 ROI 기반 OpenCV 감지를 먼저 구현한다.
- depth는 MVP 필수 조건이 아니라, RGB ROI 감지가 불안정할 때 작물/벨트 분리 보조 정보로 추가한다.
- ROI는 `belt_roi`, `entry_roi`, `middle_roi`, `exit_roi`로 나누어 벨트 전체/진입/중앙/출구 상태를 확인한다.
- 2번 카메라가 정상 작물의 벨트 진입을 확인하면 벨트 구동
- 작물 위치 추적
- 후보 중심점이 `entry -> middle -> exit` 순서로 이동하는지 확인한다.
- 작물이 벨트 끝 공용 수거 상자 방향으로 정상 이동하는지 확인
- 이상 없으면 이송 완료 처리
- 일정 시간 안에 다음 ROI로 이동하지 않거나 벨트 밖으로 벗어나면 `missing`, `stalled`, `misaligned` 같은 예외 이벤트를 기록한다.
- 다음 작물 대기

## 5. 권장 데이터 항목
작물 1개당 최소 저장 정보:
- `inspection_id`
- `timestamp`
- `crop_type`
- `quality_status`
- `camera1_image_set_path` 또는 저장 참조
- `camera2_event_log`
- `transport_target`
- `final_box`: normal_box / defect_box
- `transport_status`: delivered / dropped / missing

## 6. ROS 2 / 서버 연동 관점 초안
예시 토픽 또는 메시지 흐름:
- `/dobot/pick_status`
- `/vision/camera1/result`
- `/conveyor/object_detected`
- `/conveyor/transport_status`
- `/system/harvest_log`

웹 관제에서 우선 보여줄 값:
- 오늘 총 수확 개수
- 작물 종류별 개수
- 정상 / 불량 개수
- 현재 Dobot 상태
- 현재 컨베이어 상태
- 최근 분류 결과 로그

## 7. 예외 상황 처리 초안
- 1번 카메라 판정 실패 -> 재촬영 또는 수동 검토 큐
- Dobot 흡착 실패 -> 재시도
- 2번 카메라에서 작물 미검출 -> 벨트 정지 및 경고
- 벨트 끝 도달 전 작물 이탈/누락 -> 벨트 정지 및 알림
