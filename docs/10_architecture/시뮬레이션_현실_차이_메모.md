# SmartFarmProject 시뮬레이션-현실 구현 차이 메모

작성일: 2026-06-22
참고 이미지: `references/smartfarm simulation.png`

## 핵심 요약
이 프로젝트는 **시뮬레이션 구조**와 **현실 구현 구조**가 완전히 같지 않을 수 있다.

현재 합의한 방향은 다음과 같다.

### 1. 시뮬레이션 구조
- Dobot 수확 후 검사/분류 흐름을 포함한다.
- **3열 분류 구조**를 유지한다.
- RGB-D 2번 카메라를 사용한다.
- TurtleBot(AGV) SLAM 및 자율주행까지 포함한다.
- 트럭/물류 동선까지 시뮬레이션에서 표현한다.

### 2. 현실 구현 구조
- 1번 카메라가 메인 판정(작물 3종 + 불량 판별)
- 불량 작물은 Dobot이 **컨베이어 옆 불량품 상자**에 즉시 분리
- 정상 작물만 컨베이어에 적재
- 2번 카메라는 **컨베이어 정중앙 상단**에서 아래를 보며 흐름 확인
- 정상 작물은 **컨베이어 끝 공용 수거 상자**로 낙하
- 서보 분류기는 사용하지 않음

## simulation 이미지 기준 주요 구성요소 해석
- 좌하단 대형 영역: 농장
- 농장 인접 원형 로봇: 수확 로봇(Dobot 또는 수확 파트 로봇 개념)
- 좌측 세로 3열 박스: 토마토 / 무 / 당근 3열 분류 시뮬레이션 라인
- 좌측 하단 카메라 표기: 1번 카메라 또는 작물 검사 카메라 위치 개념
- 중앙 불량 박스: 불량품 분리 구역
- 상단 agv 박스: 터틀봇/AGV 물류 시뮬레이션 객체
- 우하단 트럭: 최종 물류 도착 지점 개념
- 우측 곡선 라인: AGV 또는 터틀봇 이동 경로 추정

## 운영 원칙
- WBS는 **시뮬레이션 기준 작업표**로 관리 가능하다.
- 다만 문서/발표에서 현실 구현 설명 시에는 시뮬레이션 구조와 혼동하지 않도록 분리해서 설명해야 한다.
- 특히 `3열 분류`는 현실 구현보다 **시뮬레이션 표현**에 가깝다는 점을 명시하는 것이 중요하다.
