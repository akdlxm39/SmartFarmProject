### Dobot 추가 확정
- 모델은 **Dobot Magician(두봇 매지션)**, 제어 방식은 **ROS 2**로 확정했다.
- 좌표는 `(x, y, z)` 순서로 기록한다.
- 수확 위치 9개를 고정 좌표로 정리했다: `(60,180,-60)`, `(100,180,-60)`, `(140,180,-60)`, `(60,220,-60)`, `(100,220,-60)`, `(140,220,-60)`, `(60,260,-60)`, `(100,260,-60)`, `(140,260,-60)`.
- 촬영 위치는 `(240,0,0)`, 불량품 상자 위치는 `(240,140,0)`, 컨베이어 시작점은 `(160,-200,-10)`로 정리했다.
- 3방향 촬영은 석션컵 360도 회전을 이용해 `0도 → 120도 → -120도 → 0도 복귀`로 수행한다.
- 흡착 성공 여부는 우선 시간 기반으로 처리한다.
- 위 내용을 `docs/Dobot_작업_메모.md`에 반영했다.

### Dobot 이동/보정 방식 추가 확정
- 석션 ON 후 흡착 대기 시간은 **2초**로 정했다.
- 촬영 신호는 Dobot이 각도 도달을 알리고, 비전 노드가 촬영 완료를 응답하는 방식으로 정했다.
- `magician_ros2`에서 homing한 위치를 `(0,0,0)` 기준 HOME/READY로 둔다.
- 다만 TCP/실제 파지점 차이 때문에 Dobot 배치가 바뀌면 실제 현재 위치를 읽어 좌표를 다시 보정하기로 했다.
- 이동은 바로 목표점으로 가지 않고, z축을 먼저 올린 뒤 x/y 이동 후 내려가는 **jump move 방식**으로 진행한다.
- 위 내용을 `docs/Dobot_작업_메모.md`, `docs/작업_결정_메모.md`, `docs/시스템_데이터_흐름_초안.md`에 반영했다.

### Dobot 임시 테스트 방식 추가 확정

- 임시 테스트에서는 각 촬영 포즈에서 2초 대기 후 다음 각도/동작으로 넘어간다.

- 작업 경로 `workspaces/지웅/vision`에 1차 소켓 통신 코드를 작성했다.

- 로컬 PC 내부 더미 영상 전송 테스트는 성공했다.
- 라즈베리파이 SSH 접속과 코드 배포, 원격 Python 컴파일은 성공했다.
- 라즈베리파이에서 카메라 목록은 `imx708_wide_noir`로 인식되었다.
- 라즈베리파이에서 mock 파일을 PC로 소켓 전송하는 테스트도 성공했다.
- 재시도 결과 실제 `rpicam-vid` 2초 촬영과 라즈베리파이→PC 소켓 전송도 성공했다.
- 수신 파일은 `1280x720` H.264로 저장되었고, remux 후 OpenCV `VideoCapture`로 첫 프레임 읽기까지 검증했다.
- PC 확인 방식은 HTTP보다 OpenCV `imshow` 창 기반을 기본으로 가져간다.
- OpenCV Qt font 경고 완화를 위해 `run_pc_receiver_opencv.sh`를 추가했고, 포그라운드 실행을 기본 사용 방식으로 정리했다.

### Dobot-비전 소켓 연동 구현
- `vision_socket_protocol.py`, `vision_capture_daemon.py`, `vision_capture_client.py`를 추가해 Dobot 모션 코드와 라즈베리파이 소켓 프로토콜을 분리했다.
- `harvest_test.py`에 `--vision-mode wait|socket|off`를 추가했고, socket 모드에서는 각도 `0/120/-120`마다 캡처 요청을 보낸다.
- 테스트 목적상 3장 캡처가 모두 성공하면 `quality_status=normal`로 판정하고, Dobot 노드는 정상 판정에 따라 컨베이어 시작점으로 이동한다.
- 판정 분기 함수도 추가해 추후 YOLO/OpenCV 결과가 `defect`를 반환하면 불량품 상자로 이동할 수 있게 했다.
- 검증 결과: vision pytest 1개, dobot pytest 3개, 기존 JPG smoke, colcon build, ros2 dry-run socket 통합 테스트 모두 통과했다.

### Dobot-비전 소켓 연결 끊김 수정
- 스크린샷 기준 라즈베리파이 클라이언트가 `connected; waiting for capture commands` 이후 `TimeoutError: timed out`로 반복 재접속하고, Dobot 노드는 `socket closed while receiving 16 bytes`로 실패하는 현상을 확인했다.
- 원인은 연결 timeout이 명령 대기 read에도 적용된 점이었다.
- `raspi_jpeg_capture_client.py` 기본 명령 대기 timeout을 무제한으로 바꾸고, 필요 시 명시적으로 `--command-idle-timeout-sec`를 줄 수 있게 했다.
- `vision_capture_daemon.py`는 Pi 소켓이 끊겼을 때 새 Pi 연결을 accept해 1회 재시도하도록 보강했다.
- 회귀 검증: 10초 이상 idle 후 캡처 성공, 기존 smoke, pytest, colcon build, ros2 dry-run socket 3장 캡처 모두 통과했다.
