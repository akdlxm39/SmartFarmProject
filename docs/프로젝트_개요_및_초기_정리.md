# SmartFarmProject 프로젝트 개요 및 초기 정리

작성일: 2026-06-22

## 1. 프로젝트 목적
SSAFY에서 학습한 임베디드 로봇/AI/Web 기술을 종합하여, **스마트팜 자동화 시스템**을 설계하고 구현한다.

핵심 방향은 다음 5개 축을 하나의 End-to-End 흐름으로 연결하는 것이다.

1. 엣지 비전 AI 검사
2. Dobot 기반 수확/분류/물주기
3. 지능형 컨베이어 제어
4. TurtleBot 자율주행 물류 운송
5. Django + Vue 기반 통합 관제

## 4. 현재까지 정리된 초기 아이디어
### 4.1 비전 AI
- 비전 모델은 **작물 종류 3종 분류 + 불량 여부(썩음 여부) 판별**을 수행한다.
- 1번 카메라는 Dobot 석션컵에 작물이 흡착된 상태에서 촬영한다.
- 1번 카메라는 작물을 **120도 간격의 3방향**에서 촬영해 다각도 정보를 확보한다.
- 2번 카메라는 컨베이어 벨트 상단에서 작물 상태를 촬영하고, 벨트 위 상태 기준으로 분류를 수행한다.
- Roboflow 기반 데이터 수집/라벨링 후 PyTorch + YOLO 계열 모델 학습을 검토한다.
- 추론 결과는 ROS 2 메시지 또는 TCP 기반으로 제어 계층에 전달한다.
- 현재 확정된 방향은 **1번 카메라가 작물 종류 분류와 불량 판별을 모두 담당**하는 것이다.
- 2번 카메라는 컨베이어 위에서 **작물 존재 확인, 위치/타이밍 확인, 컨베이어 구동 제어**를 담당하는 보조 비전/제어 카메라로 사용한다.
- 이유는 1번 카메라가 **120도 간격 3방향 촬영**이 가능해 종류 분류와 불량 판별 모두에 더 유리하기 때문이다.

### 4.2 로봇 아암(Dobot)
- Dobot은 **석션컵**을 사용해 작물을 집는다.
- Dobot은 작물을 수확한 뒤 1번 카메라 앞으로 이동해 촬영을 수행한다.
- 이때 작물은 석션컵에 흡착된 상태로 **120도 간격 3방향 촬영**이 이루어진다.
- 이후 Dobot은 작물을 컨베이어 벨트에 올려 보낸다.
- 추후 석션컵에 물뿌리개를 장착해 물 또는 비료를 분사하는 기능을 추가할 수 있다.

### 4.3 컨베이어
- 컨베이어 상단의 2번 카메라는 벨트 위에 작물이 올라왔는지 확인하고 위치/타이밍을 파악한다.
- 2번 카메라 또는 연동 센서의 판단 결과를 바탕으로 컨베이어 가동 여부를 제어한다.
- **불량품은 컨베이어에 올리지 않고 별도 상자에 담는다.**
- **정상 작물만 컨베이어를 타고 이동해 끝의 하나의 공용 상자에 담긴다.**
- 현재 설계상 2번 카메라는 **메인 판별 카메라**라기보다 **라인 흐름 확인용 보조 비전 카메라**에 가깝다.

### 4.4 자율주행 물류
- 바구니 무게 또는 적재 상태를 기준으로 배송 트리거
- TurtleBot이 맵 기반 자율주행으로 목표 위치까지 이동

### 4.5 통합 관제
- Django/MySQL로 상태 데이터 수집 및 저장
- Vue.js + Chart.js로 대시보드 구성
- 필요 시 STT/TTS 확장 가능

## 5. 지금 가장 중요한 설계 포인트
### 5.1 시스템 통신 아키텍처
- ROS 2 내부 통신과 웹 서버 간 브리지 방식 결정 필요
- 후보: rosbridge_suite, WebSocket, TCP 게이트웨이

### 5.2 엣지 리소스 제약
- 라즈베리파이에서 YOLO 추론 시 메모리/속도 이슈 예상
- 모델 경량화, 프레임 드롭, 입력 해상도 조절 전략 필요

### 5.3 물리적 상호작용 설계
- 툴 체인지 메커니즘 구체화 필요
- 컨베이어 -> TurtleBot 적재 구조 정의 필요

## 6. 초기 서브시스템 초안
- Vision Edge: 카메라 입력, 객체 판별, 결과 송신
- Dobot Controller: 로봇암 동작 제어, 툴 체인지 연동
- Conveyor Controller: 물체 감지, 모터 제어
- Mobile Robot Controller: 적재 판단, 자율주행 실행
- Backend/API: 상태 수집, 명령 전달, DB 저장
- Frontend Dashboard: 모니터링, 제어, 통계 시각화
