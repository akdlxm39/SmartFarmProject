# SmartFarmProject WBS 재검토 메모 (업데이트 반영)

작성일: 2026-06-22
대상 파일: `references/WBS.xlsx`

## 총평
업데이트된 WBS는 이전 버전보다 **현재 프로젝트 방향에 훨씬 가까워졌다.**
다만 중요한 전제가 하나 있다.

- **현실 구현 흐름**과 **시뮬레이션 흐름**이 일부 다르다.
- 사용자가 공유한 `references/smartfarm simulation.png` 기준으로, **시뮬레이션은 3열 분류 구조를 유지**할 예정이다.
- 반면 현실 구현은
  - 불량: Dobot이 별도 상자에 즉시 분리
  - 정상: 컨베이어 끝 공용 상자 1개로 이송
  구조에 가깝다.

이 전제를 두면 현재 WBS는 **시뮬레이션 기준 작업표로는 충분히 타당**하다.

## 확인 결과
### 잘 맞는 항목
- `A8 HW 세팅(2)`: 라즈베리파이 + RGB-D 카메라 + 압력센서 회로 구성
- `A10 AI 모델링`: 농작물 데이터셋 수집 및 YOLO 모델 학습
- `A11 비전 제어`: 품종/양불 판별 실시간 추론 파이프라인
- `A12 모션 제어(수확)`: 수확 -> 카메라 -> 적재/폐기 위치
- `A13 장치 제어`: RGB-D 객체 인식 기반 컨베이어 구동/정지 및 적재 로직
- `A14 자율주행`: TurtleBot SLAM + 내비게이션
- `A15 시스템 연동`: ROS Bridge 기반 서버 연동
- `A19 통합 테스트`: 수확부터 비전 검사, 분류, 터틀봇 배송까지 End-to-End 검증

### 시뮬레이션 기준으로 허용 가능한 차이
- `A4 로봇 시뮬레이션`의 `3열 분류` 표현은 **현실 구현 기준으로는 다르지만**, 시뮬레이션이 별도 구조를 가진다는 전제에서는 유지 가능하다.
- 즉, 이 항목은 수정 필수는 아니다.
- 오히려 문서에서 `시뮬레이션 구조와 현실 구현 구조가 다름`만 명시하면 된다.

## 있으면 더 좋은 보완 포인트
이건 필수 수정까진 아니지만, 있으면 더 좋다.

### 1. 2번 카메라 시야/배치가 WBS에 더 드러나면 좋음
현재 대화 기준:
- 2번 카메라 = 컨베이어 정중앙 상단에서 아래 방향
- 역할 = 작물 존재/위치/흐름 확인

이 내용이 `A13 장치 제어`나 `A8 HW 세팅(2)` 상세에 조금 더 드러나면 팀원끼리 이해가 빨라질 수 있다.

예:
- `RGB-D 상단 카메라 기반 컨베이어 흐름 감지 및 적재 로직`

### 2. 물류 연동(A16)은 MVP 포함 여부를 팀 내부에서 일정상 다시 체크
사용자 기준으로 TurtleBot은 MVP 포함이 맞다.
다만 3인 프로젝트에서는 난도가 높은 편이라,
- SLAM/주행 자체를 먼저 성공시키고
- 압력센서 연동은 단계적으로 붙이는 순서
로 가면 더 안정적일 수 있다.

즉, WBS 항목은 유지하되 실제 일정에서는 뒤로 배치하는 것이 좋다.

## 결론
현재 `references/WBS.xlsx`는 **시뮬레이션 기준으로 거의 그대로 가도 된다.**

핵심은 WBS를 억지로 현실 구현과 100% 같게 맞추는 것이 아니라,
- **시뮬레이션은 3열 분류 구조 유지**
- **현실 구현은 불량 즉시 분리 + 정상 공용 상자 이송**
처럼 두 흐름이 다르다는 점을 팀이 명확히 알고 가는 것이다.

즉, 지금은 항목을 더 고치기보다,
**"시뮬레이션 구조와 현실 구현 구조는 다를 수 있다"**는 점만 별도 메모로 남기면 충분하다.
