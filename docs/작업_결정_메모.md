## 결정 주제
Dobot Magician ROS 2 제어 기준 및 좌표/촬영 방식

## 현재 결정
- Dobot 모델은 **Dobot Magician(두봇 매지션)**이다.
- SDK/제어 방식은 **ROS 2**로 진행한다.
- 좌표는 `(x, y, z)` 순서로 관리한다.
- 수확 위치는 고정 9개 좌표를 사용한다.
- 촬영 위치는 `(240, 0, 0)`이다.
- 불량품 상자 위치는 `(240, 140, 0)`이며, 위에서 석션 OFF로 떨어뜨린다.
- 컨베이어 시작점 위치는 `(160, -200, -10)`이다.
- 3방향 촬영은 석션컵 360도 회전을 이용해 `0도 → 120도 → -120도 → 0도 복귀` 순서로 수행한다.
- 흡착 성공 여부는 센서 판정보다 **시간 기반 대기**로 먼저 구현한다.

## 영향
- Dobot 작업의 우선 구현 단위는 `harvest_index -> 촬영 3방향 -> 임의 정상/불량 분기` 테스트가 된다.
- 이후 세부 구현에서 추가로 정할 것은 흡착 대기 시간, 촬영 완료 신호 방식, HOME/READY 좌표, 안전 이동 waypoint다.

## 결정 주제
Dobot 흡착 대기, 촬영 신호, HOME/좌표 보정, 안전 이동 방식

## 현재 결정
- 석션 ON 후 흡착 대기 시간은 **2초**로 한다.
- 촬영 완료 신호는 `Dobot 각도 도달 알림 -> 비전 노드 촬영 완료 응답` 방식으로 진행한다.
- HOME/READY 기준은 `magician_ros2`에서 homing한 위치를 `(0, 0, 0)`으로 본다.
- 단, ROS 2 통신만으로는 TCP/실제 파지점 위치를 정확히 알기 어려우므로, Dobot 배치가 바뀌면 실제 위치를 읽어 좌표를 다시 보정한다.
- 이동은 바로 목표점으로 직선 이동하지 않고, **jump move 방식**으로 z축을 먼저 올린 뒤 x/y 이동 후 목표 z로 내려간다.

## 영향
- Dobot 좌표표는 현재 기준 좌표로 유지하되, 최종 장비 배치 후 현재 pose 읽기 기반으로 갱신해야 한다.
- 다음 구현 단위는 `harvest_index -> jump move -> 석션 2초 대기 -> 3방향 촬영 -> 임의 정상/불량 분기` 테스트가 된다.
- 아직 정할 값은 jump move 안전 z 높이와 비전 결과 대기 timeout이다.

---

## 결정 주제
Dobot jump move 안전 높이와 비전 미연동 임시 처리

## 현재 결정
- jump move 안전 높이는 **z=100mm**로 한다.
- 현재 비전 파트가 바로 붙어 있지 않으므로, 촬영 완료 신호 대기와 비전 판정 결과 대기는 일단 주석 처리한다.
- 임시 테스트에서는 각 촬영 포즈에 도달한 뒤 **2초 대기**하고 다음 각도/동작으로 넘어간다.

## 영향
- Dobot 단독 테스트 시퀀스는 `harvest_index -> z=100mm jump move -> 석션 2초 -> 0/120/-120도 각도별 2초 대기 -> 테스트 분기 -> 홈 복귀`가 된다.
- 비전 결과 timeout은 지금 정하지 않고, 비전 노드가 붙은 뒤 다시 결정한다.

## 결정 주제
Dobot 실측 위치 재보정 결과 반영

## 현재 결정
- 위치 재보정 결과 파일은 `workspaces/지웅/ros2_ws/src/dobot_control_pkg/config/dobot_positions_latest.json`을 기준으로 사용한다.
- `/dobot_TCP`에서 저장된 좌표는 meter 단위이다.
- 수확 위치 9개는 좌하단 `(0.0511, 0.2006, -0.0509)`과 우상단 `(0.1077, 0.1312, -0.0509)` 범위를 3x3으로 분할한 값을 사용한다.
- 촬영 위치는 `(0.2352, -0.0028, 0.0673)`이다.
- 불량품 상자 위치는 `(0.1782, 0.1138, 0.0336)`이다.
- 컨베이어 시작점 위치는 `(0.1041, -0.1499, 0.0183)`이다.
- 이후 제어 코드가 mm 단위를 요구하면 위 좌표에 1000을 곱해 반영한다.

## 영향
- 기존 임시 좌표 `(240, 0, 0)`, `(240, 140, 0)`, `(160, -200, -10)`은 실측 결과로 대체할 후보가 생겼다.
- 다음 Dobot 제어 코드 작성/수정 시 JSON 결과 파일을 읽거나, 같은 값으로 좌표 상수를 갱신한다.

## 결정 주제
Dobot 단일 수확 테스트 시퀀스

## 현재 결정
- 첫 테스트 노드는 `harvest_test`로 작성한다.
- 사용자가 수확 인덱스를 직접 입력해 대상 위치를 선택한다.
- 수확 인덱스는 **x 내림차순 -> y 내림차순**으로 다시 부여한다.
- 수확 후 촬영 위치에서 `0도 -> 120도 -> -120도 -> 0도 복귀`를 수행하고, 각도별 2초 대기한다.
- 비전 결과는 아직 붙이지 않고 정상으로 임시 고정한다.
- 정상 분류된 작물은 컨베이어 시작점으로 이동한 뒤 석션 OFF한다.

## 영향
- Dobot 단독 테스트는 수확/흡착/촬영 회전/정상 컨베이어 적재까지 한 번에 확인하는 최소 시퀀스가 된다.
- 비전 연동이 붙으면 각도별 2초 대기 부분을 촬영 완료/판정 결과 대기로 교체한다.

## 결정 주제
Dobot 수확 위치 9점 직접 보정과 촬영 회전 보정

## 현재 결정
- `240도` 회전 goal이 reject되었으므로 촬영 회전은 `0도 -> 120도 -> -120도 -> 0도 복귀`로 사용한다.
- 수확 위치 z는 **-50mm**로 고정한다.
- 수확 위치는 좌하단/우상단 2점 기반 분할을 쓰지 않고, 9개 위치를 모두 직접 찍는다.
- 수확 위치 9점 재보정 노드는 `calibrate_harvest_positions`로 작성한다.
- 이 노드는 기존 좌표 파일에서 카메라/불량상자/컨베이어 위치는 유지하고 수확 위치 9개만 교체한다.

## 영향
- `harvest_test`의 기본 수확 z는 `-50mm`가 된다.
- 수확 위치를 다시 캡처한 뒤 `dobot_positions_latest.json`만 갱신하면 기존 테스트 노드가 새 위치를 읽어 동작한다.

## 결정 주제
Dobot 비전 소켓 연동 구조와 임시 판정 정책

## 현재 결정
- Dobot 모션 노드 `harvest_test.py`에는 라즈베리파이 소켓 서버/패킷 저장 로직을 넣지 않는다.
- PC 쪽 `vision_capture_daemon.py`가 라즈베리파이 연결, JPG 패킷 수신, 파일 저장, 임시 판정 반환을 담당한다.
- Dobot 쪽은 `vision_capture_client.py`로 로컬 데몬에 캡처 요청만 보낸다.
- 촬영 각도 `0/120/-120`의 JPG 3장을 모두 받으면 현재 테스트 목적상 `quality_status=normal`로 판정한다.
- Dobot 노드는 판정 결과가 `normal`이면 컨베이어 시작점, `defect`이면 불량품 상자로 이동한다.

## 영향
- 추후 OpenCV/YOLO 추론은 `vision_capture_daemon.py`에서 `quality_status`를 산출하는 방식으로 확장한다.
- `harvest_test.py`는 `--vision-mode wait|socket|off`로 Dobot 단독 테스트와 비전 통합 테스트를 모두 지원한다.
- 저장 경로는 `workspaces/지웅/vision/incoming_jpegs/<sequence_id>/angle_*.jpg`를 기본으로 한다.
