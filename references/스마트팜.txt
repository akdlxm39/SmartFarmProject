제시해주신 스마트팜 프로젝트는 로봇 제어, 비전 AI, 자율주행, 그리고 웹 관제까지 이어지는 훌륭한 End-to-End 시스템입니다. 사진으로 첨부해주신 기술 스택을 최대한 활용하여, 전체 프로세스를 5개의 핵심 Task로 분류하고 각 파트별로 적용할 기술과 구체화해야 할 포인트들을 정리해 드리겠습니다.

---

### 🛠️ 스마트팜 프로젝트 Task 분류 및 기술 스택 매핑

#### **Task 1: 엣지 컴퓨팅 기반 비전 AI 검사 (품종 분류 및 양불 판별)**

라즈베리파이와 카메라 센서를 이용해 수확한 작물의 상태를 즉각적으로 분석하는 단계입니다.

* **기술 스택:** Raspbian, Python, PyTorch, YOLO, roboflow, OpenCV
* **구현 내용:**
* roboflow를 활용해 작물의 품종과 불량 상태(형태 이상, 변색 등)에 대한 이미지 데이터를 수집하고 라벨링합니다.
* PyTorch와 YOLO 모델을 학습시켜 라즈베리파이(Edge Device)에 탑재합니다.
* OpenCV로 카메라 프레임을 제어하고, 추론된 결과(품종, 양/불량)를 ROS 2 메시지나 TCP 통신을 통해 메인 서버 및 Dobot 제어 노드로 전송합니다.



#### **Task 2: 로봇 아암(Dobot) 작업 및 모션 제어**

작물 수확, 검사 위치로의 이동, 분류(폐기/컨베이어 적재), 그리고 툴 체인지를 통한 물주기 작업을 수행합니다.

* **기술 스택:** ROS 2, C/C++, Python, roboDK, RS232/TCP, Onshape, 3D printing
* **구현 내용:**
* roboDK를 활용해 Dobot의 수확 지점, 카메라 지점, 폐기 지점, 컨베이어 3열 적재 지점에 대한 기구학적 모션 플래닝을 사전에 검증합니다.
* ROS 2 환경에서 Dobot 제어 노드를 C++나 Python으로 작성하여, AI의 판별 결과에 따라 정확한 위치(3열 중 하나)에 작물을 내려놓도록 분기 처리합니다.
* **물주기 기능:** 진공 그리퍼와 물뿌리개를 교체할 수 있는 엔드이펙터 마운트를 Onshape로 설계하고 3D 프린터로 제작하여 물리적인 툴 체인지 기능을 구현합니다.



#### **Task 3: 지능형 컨베이어 벨트 시스템**

비전 센서를 통해 컨베이어의 작동을 자동화하고 자원을 효율적으로 관리합니다.

* **기술 스택:** ROS 2, OpenCV, Numpy, RGB-D 카메라 제어 (또는 URDF 연동)
* **구현 내용:**
* RGB-D 카메라로 컨베이어 벨트 위를 감시하며, Numpy와 OpenCV를 이용해 Depth 정보와 RGB 정보를 통합 처리하여 물체의 존재 여부 및 위치를 정밀하게 파악합니다.
* 객체가 인식되면 Modbus 또는 내부 통신을 통해 컨베이어 모터를 구동하고, 빈 궤도임이 확인되면 동작을 정지시킵니다.



#### **Task 4: 자율주행 모바일 로봇 물류 운송 (TurtleBot)**

적재가 완료된 바구니를 목표 지점까지 배송합니다.

* **기술 스택:** Ubuntu, ROS 2, RViz, Gazebo, SLAM
* **구현 내용:**
* 바구니 하단의 압력 센서 데이터를 읽어 임계치(무게)를 초과하면 TurtleBot에 배송 명령을 내립니다.
* TurtleBot은 SLAM을 통해 구축된 농장 맵 내에서 목적지까지 자율 주행합니다. 실제 주행 전 Gazebo와 RViz를 통해 가상 환경에서 주행 알고리즘과 장애물 회피 성능을 시뮬레이션합니다.



#### **Task 5: 통합 관제 서버 및 Web UI**

전체 하드웨어의 상태를 관리하고 사용자와 상호작용하는 대시보드입니다.

* **기술 스택:** Django / MySQL (Backend), Vue.js, Vue router, Chart.js (Front), Server/Client 통신
* **구현 내용:**
* Django 백엔드에서 각 로봇(Dobot, TurtleBot)과 AI 엣지 디바이스의 상태 데이터를 수집하여 MySQL에 저장합니다.
* Vue.js 기반의 프론트엔드에서 수확량, 불량률 등의 통계를 Chart.js로 시각화합니다.
* 사용자가 '물 주기' 버튼을 누르면 Django 서버를 거쳐 Dobot 제어 노드로 명령이 하달되는 양방향 통신 구조를 구축합니다.
* *(추가 활용)* 남는 기술 스택인 **STT/TTS**를 웹 UI에 적용하여, 사용자가 음성으로 "물 주기 시작해"라고 명령(STT)하거나, 서버에서 "불량품이 과다 발생했습니다"라고 음성 알림(TTS)을 주도록 구성할 수 있습니다.

### 🔍 더 구체화해야 할 핵심 포인트

성공적인 프로젝트를 위해 설계 단계에서 다음 세 가지를 반드시 짚고 넘어가야 합니다.

1. **전체 시스템의 통신 아키텍처 설계 (Data Flow)**
* ROS 2 시스템(센서, 모터 제어)과 웹 서버(Django) 간의 브릿지를 어떻게 구성할 것인지 명확히 해야 합니다. `rosbridge_suite`나 WebSocket을 활용하여 ROS 메시지를 JSON 형태로 변환해 웹과 실시간으로 주고받는 구조를 설계해야 지연 없는 관제가 가능합니다.


2. **엣지 환경의 리소스 및 메모리 최적화 (OOM 대비)**
* 라즈베리파이 환경에서 무거운 YOLO 모델을 그대로 돌리거나, 여러 카메라 센서의 스트리밍 데이터를 동시에 처리하다 보면 메모리 초과(OOM)가 발생하여 시스템이 다운될 수 있습니다. AI 모델을 경량화(YOLOv8n 등)하거나, 비전 처리 시 데이터 슬라이싱 및 프레임 드롭 기법을 적용하여 시스템 부하를 제어하는 로직이 반드시 포함되어야 합니다.


3. **물리적 상호작용(Physical Interaction)의 디테일**
* **툴 체인지 메커니즘:** Dobot이 자력이나 기계적 결합을 통해 툴을 어떻게 스스로 교체할 것인지 구체적인 기구 설계가 필요합니다.
* **바구니 상차 문제:** 컨베이어 끝에서 터틀봇으로 바구니가 어떻게 옮겨지는지(터틀봇이 컨베이어 바로 아래로 들어가서 바구니가 떨어지는 구조인지, 별도의 푸셔(Pusher)가 있는지) 물리적인 동선과 기계적 설계가 명확해야 합니다.
